Good questions

Wondering about something?

Is Carrier really free?

Completely. No price tag, no subscription, no ads, no "premium" version waiting around the corner. Carrier is open source under the MIT license — made because we missed having Messenger on the desktop.

Do I need a new account?

No — you sign in with your usual account, and all your chats are just there. Anything you send in Carrier shows up on your phone too, and the other way around.

Is this an official Meta app?

No. Meta discontinued their desktop app, so Carrier is an independent, open-source replacement. It shows you the real Messenger — the same one from your browser — in a clean window of its own, with the Facebook clutter tidied away.

Is it safe to use?

Carrier adds zero tracking of its own — it even blocks Facebook's telemetry by default. The Mac version is signed and notarized by Apple, updates are verified before they install, and the complete source code is public for anyone to inspect. Your messages still travel through Messenger itself, so Meta's usual terms apply there.

Do voice and video calls work?

Yes — it's the full Messenger, so calls, group chats, reactions, photos, and everything else work like you'd expect.

How is the whole app only 3 MB?

Most chat apps ship an entire hidden web browser inside. Carrier uses the one your computer already has — so it downloads in seconds, starts in a blink, and goes easy on your battery.

What if something stops working?

Facebook redecorates now and then, and once in a while that nudges something out of place. Carrier keeps itself up to date and fixes usually arrive quickly — and you can always tell us what broke.

For the curious 🤓The slightly more technical answers

How does it work under the hood?

Carrier is a small Rust shell built on Tauri 2 that opens the official Messenger web app in your operating system's own WebView. At page load it injects a focused stylesheet and typed scripts that hide Facebook's surrounding chrome and add the desktop features — notifications, badges, shortcuts, themes. No bundled browser engine, no Electron.

What can the injected code see?

Messenger content is processed only on your machine, only where a feature needs it — turning a new message into a native notification, counting unread chats for the badge, blurring names for screen sharing. Carrier has no servers and phones home to no one; the only network calls it adds are update checks against GitHub Releases. Off-site links open in your browser with Facebook's tracking redirect stripped first.

How do automatic updates work?

Carrier checks GitHub Releases and cryptographically verifies every update's signature before installing it. On macOS the downloads are also Developer ID signed and notarized by Apple, so Gatekeeper vouches for what you run.

Why is the Linux AppImage bigger than the installers?

The .deb and .rpm packages are about 4 MB because they use your distro's WebKitGTK. The AppImage has to carry its own copy to run anywhere, which is why it weighs around 90 MB. If your package manager is an option, prefer it.

Is there a Homebrew cask?

Yes — brew install --cask kristofferR/tap/carrier, and the same cask token works on macOS and Linux. Already dragged Carrier.app into /Applications? Adopt it with brew install --cask --adopt kristofferR/tap/carrier.

Can I build it myself?

Of course. With Rust, Bun, and the Tauri prerequisites installed: clone the repo, then bun install and bun run build. The injected scripts are typed TypeScript under inject/src/.
